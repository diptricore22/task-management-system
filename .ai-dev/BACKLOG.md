## 📌 Rules for Backlog Management

1. **Never delete an idea** — move to "Rejected" instead
2. **Always capture the source** of the idea (client, dev insight, user feedback, AI suggestion)
3. **Review at the start of each new phase** before planning
4. **Effort sizing:** S = < 1 day, M = 1-3 days, L = 4-7 days, XL = 1+ week
5. **Scope changes must document the reason** — future-you will need to know why
